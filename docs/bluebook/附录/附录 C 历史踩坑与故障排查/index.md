# 附录 C 历史踩坑与故障排查

本附录只记录可迁移的方法，不展示内部项目、系统、人员和数据。

| 现象 | 常见原因 | 安全处理 |
|---|---|---|
| AI 写得很顺，但事实不对 | 输入缺口被模型补全 | 先建事实与来源清单；无依据内容标记待确认 |
| 一次批量处理后错误很多 | 没有先做最小样本 | 先处理一页、三条或少量文件，验收后再扩大 |
| 文件整理后找不到原件 | 直接移动、覆盖或删除 | 原始目录只读，先输出操作预览和冲突清单 |
| “最终版”有多个 | 只看文件名，没有审批依据 | 用确认、签署、日期和版本关系判断，状态不明先保留 |
| 自动化没有输出 | 任务仍在运行、路径错误或静默失败 | 检查进程、日志、输入和输出目录，不盲目重复启动 |
| 自动化重复写入 | 没有幂等标识或同类任务预检 | 运行前检查批次和目标记录，重复项跳过并记录 |
| 信息摘要与原文不一致 | 只使用二手摘要或未核验来源 | 回到原始来源核对主体、日期和关键事实 |
| 数据图表结论偏差 | 口径不一致、过滤小样本或混淆因果 | 固化口径，显示样本和限制，结论绑定证据 |
| 知识库越来越大却找不到 | 只有收藏，没有来源页和使用目的 | 一项来源一条记录，知识点关联具体任务 |
| 远程任务权限不足 | 目录、账号或网络边界未预检 | 在本地先测试最小权限，远程不扩大授权 |
| 多 Agent 输出互相矛盾 | 角色边界和共享字段不一致 | 定义输入输出契约，冲突并列交人工确认 |
| 构建成功但网站仍是旧内容 | 只更新源码，未同步公开构建仓库 | 构建后检查部署提交和线上目标页面 |

## 通用排查顺序

1. 先确认目标和预期结果。
2. 再检查输入、路径、版本和权限。
3. 查看任务状态、日志和中间产物。
4. 用最小样本复现问题。
5. 只修改一个变量并重新验证。
6. 修复后补充检查表或自动测试，避免重复踩坑。

## 可复制故障报告

```text
任务目标：
预期结果：
实际现象：
首次出现时间：
输入和工作目录：

错误信息或日志位置：
已经尝试的操作：
禁止执行的操作：删除、覆盖、扩大权限、重复批量启动
```
