## A.3 高级互动PPT 生成

```text
一、布局结构三段法** - HTML结构必须清晰：容器→幻灯片→内容区 - 每个幻灯片必须用scroll-snap固定，防止滑动中断 - 所有元素必须使用相对单位（vw/vh），绝不用像素 - 关键内容永远放在视觉黄金分割点（左上角起始点38.2%处）
二、视觉层次四要素** - 背景必须极简：纯黑/深色+微妙纹理（opacity不超过0.2） - 标题必须醒目：字号是正文的2.5倍+强对比色+粗体 - 强调元素必须突出：渐变色文字+大胆边框+悬停变化 - 动画必须克制：最多4种效果，必须有延迟差（0.2s递增）
三、交互体验五步法** - 导航必须直观：底部居中小圆点+左右箭头 - 翻页必须流畅：使用scrollIntoView而非硬切换 - 响应必须灵敏：所有可点击元素hover效果延迟不超过0.1s - 动画必须重置：每次翻页都要重新触发当前页动画 - 控制必须全面：同时支持点击、键盘和触摸滑动 ```html <!-- 核心布局结构 --> <div class="slide-container"> <div class="slide"> <div class="slide-content"> <!-- 内容放这里 --> </div> </div> </div> <!-- 核心样式 --> <style> .slide-container { width: 100vw; height: 100vh; scroll-snap-type: x mandatory; display: flex; overflow-x: hidden; } .slide { flex: 0 0 100vw; height: 100vh; scroll-snap-align: start; position: relative; } /* 动画效果 */ @keyframes fadeIn { from { opacity: 0; transform: translateY(30px); } to { opacity: 1; transform: translateY(0); } } .fade-in { opacity: 0; animation: fadeIn 0.8s ease-out forwards; } </style> <!-- 核心交互脚本 --> <script> // 滑动到指定页面 function scrollToSlide(index) { slides[index].scrollIntoView({behavior: 'smooth'}); resetAnimations(index); } // 重置动画 function resetAnimations(slideIndex) { const animElements = slides[slideIndex].querySelectorAll('.animated'); animElements.forEach(el => { el.style.animation = 'none'; el.offsetHeight; el.style.animation = ''; }); } </script> ``` 
优化效果检查表 | 代码要素 | 完成状态 | 实现方式 | |---------|---------|---------| | 响应式布局 | ✓ | 使用vw/vh相对单位，适应各种屏幕 | | 滑动机制 | ✓ | 使用scroll-snap-type实现平滑翻页 | | 动画系统 | ✓ | 使用CSS动画+JS重置机制实现动态效果 | | 交互控制 | ✓ | 实现点击、键盘、触摸三种控制方式 | | 视觉层次 | ✓ | 通过大小、颜色、动画时间差创造层次感 | | 代码结构 | ✓ | HTML结构清晰，CSS模块化，JS功能独立 |
```

## A.4 会议纪要

```text
请根据以下会议内容生成结构化纪要:

{{meeting_content}}

请按以下格式输出:

## 会议基本信息
- 会议主题:
- 参会人员:
- 会议时间:
- 会议时长:

## 核心议题
提取3-5个主要议题，每个用一句话概括

## 讨论详情
对每个议题展开说明:
- 议题名称
- 主要观点(控制在150字以内)
- 不同意见
- 达成的共识

## 决策事项
所有确定的决策，格式:
【已决策】决策内容 | 执行要求 | 负责人

## 待办任务
格式:任务内容 | 责任人 | 截止时间

## 遗留问题
【待讨论】问题描述 | 涉及方
```

## A.5 行业调研

```text
# 【你的问题描述】
请用户提供研究主题、目标、关注点等信息。

📌 模板结构

<研究背景 /context>
请说明背景信息，即"我为什么要做这个研究"。

<研究需求 /goals>
请明确研究主题、研究目标、需要关注的地方；也即"这个研究是什么、关注什么"。

🔍 通用要求 /requirements：

语言语法：使用中文撰写，但仅采纳英文资料（因为互联网上英文资料数量和质量通常是最好的）。

读者对象：根据指向普通读者（自我提升、终身学习、思维训练）的需求进行写作，兼顾可读性与学术性。

报告长度：建议深度够、结构清晰，长度不低于20000字上下封顶。一切以深入全面解读为目标，拒绝浅尝辄止。

参考资料：请参考 Wikipedia、相关书籍、学术和科普期刊网站、权威媒体和杂志刊物内容；优先引用公开信息。

可视化：鼓励在研究报告中采用表格、图表等可视化形式来辅助表达和加深理解。
```

## A.7 沟通汇报

```text
方案1：问题-分析-改进-承诺
- 坦诚说明业绩未达标的现状
- 分析客观原因和主观原因
- 提出具体的改进措施
- 承诺下季度的目标

方案2：亮点-不足-学习-计划
- 先说工作中的亮点（即使小也要说）
- 承认不足之处
- 表达学习意愿
- 列出下一步行动计划

方案3：对比-反思-成长-请求
- 与往年同期对比（如果有改善）
- 深刻反思问题所在
- 强调从失败中学到的成长
- 请求领导给予支持和机会

方案4：价值-潜力-调整-信心
- 强调我为团队带来的其他价值
- 说明我的潜力和能力
- 调整工作方法和重点
- 表达对未来的信心
```
