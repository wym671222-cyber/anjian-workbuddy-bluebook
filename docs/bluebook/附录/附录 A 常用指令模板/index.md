# 附录 A 常用指令模板

下面的模板先限定输入、输出、验收和禁止动作，再让 WorkBuddy 执行。使用真实材料前，请按附录 D 完成脱敏和权限检查。

## A.1 澄清模糊任务

```text
我的初步需求是：{需求}。
请先不要执行，用不超过 8 个问题帮助我明确目标、读者、输入、输出、截止时间、验收标准和禁止动作。
根据回答生成任务卡，并建议一个最小可验收样本。
```

## A.2 文件整理

```text
请只读检查当前目录，生成文件清单、版本关系、异常和建议目录。
先输出拟操作预览；没有我确认，不移动、重命名、删除或覆盖文件。
```

## A.3 表格分析

```text
请先报告表格字段、数据量、空值和异常，不修改原文件。
等我确认指标口径后，再生成处理表、汇总表和图表。
每条结论注明证据、样本量和限制，不展示个人信息。
```

## A.4 汇报材料

```text
请根据已确认材料生成 10 页以内的汇报大纲。
先列每页要回答的问题、关键证据和建议图表，等我确认后再生成文件。
每页只表达一个核心观点；不得新增未经证实的数字。
输出逐页内容清单、跨文件数字核对表和待确认问题。
```

## A.5 会议纪要与待办

```text
请根据已授权使用的会议材料生成结构化纪要草稿。

输出结构：
## 会议基本信息
- 会议主题、时间、参会角色、材料范围

## 核心议题
- 每个议题用一句话概括

## 讨论情况
- 主要观点、不同意见、已形成共识、仍待确认事项

## 确认结论
- 只记录材料能够证明已经确认的结论

## 待办任务
- 任务、责任角色、协作角色、截止时间、验收标准、状态、来源

## 遗留问题
- 问题、缺失材料、需要确认的角色

不能确认的责任人、日期和结论统一写“待确认”，不要自行补全。
先输出预览，不自动发送或写入外部系统。
```

## A.6 多文档检索

```text
请先建立文件清单并记录名称、版本、日期、来源和状态。
围绕问题提取候选条款，标注文件、章节和页码。
把原文、摘要和推断分开，不判断文件效力或给出权威解释。
```

## A.7 公开资料研究

```text
研究主题：{主题}
研究背景：{为什么研究}
研究目标：{需要回答的问题}
时间和地域范围：{范围}
允许来源：政府、权威机构、学术出版物、企业公开披露等
排除来源：{来源类型}
输出对象与篇幅：{读者和长度}

请先提交研究提纲、检索词和来源选择标准；
等我确认后再开展研究。
事实、引用和分析判断必须分开；每项关键事实附原始链接与日期；
对互相冲突的资料并列说明，不为满足篇幅重复内容或编造结论。
```

## A.8 项目资料研判

```text
请建立资料目录、事实表、证据矩阵、风险清单和待核实问题。
区分事实、第三方观点、假设和建议，不作投资、审批或专业结论。
不得修改原始材料。
```

## A.9 自动化预检

```text
运行前检查输入、权限、输出冲突和同类任务状态。
先用脱敏小样本运行，并记录每一步状态。
没有明确人工确认，不执行外发、删除、覆盖、审批和业务写入。
```

## A.10 互动演示网页

```text
请把已确认的汇报大纲制作成可在浏览器打开的互动演示网页。

要求：
- 一屏一页，标题、正文和图表层次清楚；
- 同时支持按钮、键盘和触摸切换；
- 动画克制，并支持“减少动态效果”系统设置；
- 使用响应式布局，避免文字溢出和元素重叠；
- 保留页码、导航、打印或导出说明；
- 先生成两页样稿，桌面和移动端验收后再完成全部页面。

注意：互动网页不是 PPTX 文件。若最终需要 PPTX，请另行生成并独立验收。
```

## A.11 沟通汇报

```text
请把以下工作情况整理成“事实—原因—行动—需要支持”的汇报草稿。

已完成事实：{事实与依据}
存在差距：{与目标的差距}
客观原因：{有证据的原因}
主观改进：{可以控制的行动}
下一步计划：{任务、时间和验收}
需要支持：{明确请求}

不要夸大成果，不把计划写成完成，不推卸责任。
先列事实和待确认问题，再生成汇报草稿。
```
