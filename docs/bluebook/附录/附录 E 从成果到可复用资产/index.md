# 附录 E 从成果到可复用资产

一次成功任务只有在能够复现、检查和维护时，才值得沉淀。

## 五级沉淀路径

| 级别 | 适合保存什么 | 进入下一级的条件 |
|---|---|---|
| 结果 | 通过验收的文件和记录 | 事实、版本和验收清楚 |
| 模板 | 稳定的结构、字段和检查表 | 至少两次任务结构相似 |
| 方法卡 | 适用场景、输入、步骤、边界和验收 | 方法可以由另一人复现 |
| Skill | 可重复执行的方法与配套资源 | 异常、测试和人工闸门明确 |
| 自动化 | 定期或批量执行的可靠任务 | 输入稳定、可幂等、可停止和回退 |

## 复盘问题

1. 本次真正解决了什么问题。
2. 哪些输入是必需的，哪些只是偶然出现。
3. 哪些步骤稳定，哪些依赖人工判断。
4. 最容易出错的环节是什么。
5. 验收人如何判断结果合格。
6. 哪些内容不能复用或公开。
7. 下次可以先复用模板、方法卡还是 Skill。

### 可复制沉淀模板

```text
已验收成果：{成果}
解决的问题：{问题}
必要输入：{输入}
稳定步骤：{步骤}
人工判断：{判断点}
常见错误：{错误}
验收标准：{标准}
安全边界：{边界}

请判断本次只应保存结果，还是可以升级为模板、方法卡、Skill或自动化。
说明理由、仍缺少的验证和下一次复现计划。
```

## 安全边界

- 不把一次偶然成功直接固化为正式方法。
- 不在模板和 Skill 中嵌入真实业务数据、密钥和个人信息。
- 不把需要专业判断和审批的步骤改成无人值守。
- 每项资产记录版本、日期、适用范围和维护责任。
