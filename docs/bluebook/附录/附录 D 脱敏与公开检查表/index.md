# 附录 D 脱敏与公开检查表

公开不是简单删除姓名。文件名、项目结构、部门、数字、时间组合和截图界面都可能让人反推出真实事项。

## 四级检查

### 1. 来源权限

- 材料是否允许用于本任务。
- 是否允许复制、上传、改写和公开。
- 是否包含第三方版权或保密要求。

### 2. 内容脱敏

- 删除姓名、手机号、证件号、邮箱和账号。
- 删除真实部门、项目名称、合同编号、金额和未公开指标。
- 替换时间、地点、样本量和业务细节的可识别组合。
- 截图检查标题栏、文件路径、通知、头像和隐藏工作表。

### 3. 事实边界

- 通用方法不得写成公司已经正式采用的流程。
- 合成样例明确说明为虚构。
- 模型推断、建议和原始事实分开。
- 公司事实必须有批准来源和确认记录。

### 4. 发布动作

- 检查目标网站、接收对象和访问范围。
- 构建产物再次进行敏感词和本机路径扫描。
- 由有权限人员确认后发布。
- 保留发布版本、确认记录和撤回办法。

## 可复制公开检查提示

```text
请只检查这份待公开材料，不改写原文件。

检查：个人信息、真实部门、项目名称、金额、合同信息、内部系统、
本机路径、账号密钥、截图残留、未经证实的公司事实、
把通用方法写成正式流程的表述。

输出问题位置、风险类型、修改建议和需要人工确认的人。
不要根据上下文补造脱敏内容，不要自动发布。
```

## 验收标准

- 公开材料不能还原到具体人员、项目或内部系统。
- 示例的名称、数字、文件和截图均为合成内容。
- 高风险事实有来源与确认记录。
- 发布前后都可以定位版本并执行撤回。
