# 第 25 章 多 Agent 分工、汇总与复核

多 Agent 不是“多开几个聊天窗口”，而是把一个复杂任务拆成边界清楚的角色，让结果可以相互检查，最后仍由一个责任人整合。

## 什么时候值得使用

- 材料量大，需要并行整理不同类别。
- 任务包含事实核验、结构设计、风险检查等不同能力。
- 单一 Agent 容易遗漏反向证据或混淆角色。
- 中间成果可以单独验收和合并。

简单任务不要强行使用多 Agent。分工和汇总的成本可能高于直接完成。

## 合成场景

为一份虚构的“办公区域改造方案”准备评审材料。所有输入均为合成样本，可拆为五个角色：

| 角色 | 负责什么 | 不负责什么 |
|---|---|---|
| 材料整理 Agent | 建目录、识别版本与缺口 | 不评价方案 |
| 事实核验 Agent | 核对数字、日期和出处 | 不补造缺失事实 |
| 结构 Agent | 组织汇报逻辑和页面大纲 | 不改变事实 |
| 风险检查 Agent | 查找假设、矛盾和待确认项 | 不确定风险等级 |
| 汇总 Agent | 合并结果、处理重复和冲突 | 不替代责任人定稿 |

## 设计步骤

1. 先写总任务目标和最终验收标准。
2. 按输入或专业边界拆角色，不按“看起来厉害”的职位拆分。
3. 为每个角色规定输入、输出、禁止动作和完成条件。
4. 设计共享字段，例如来源编号、事实状态、问题编号和置信状态。
5. 让角色输出冲突而不是相互覆盖。
6. 汇总后由责任人确认事实、取舍和最终结论。

### 可复制设计模板

```text
复杂任务：{任务}
最终输出：{输出}
总体验收：{标准}

请先判断是否真的需要多 Agent。
如需要，请为每个角色定义：
- 角色目标；
- 可读取的输入；
- 必须输出的字段；
- 禁止动作；
- 完成条件；
- 与其他角色的交接方式。

所有角色必须保留来源编号；
发现冲突时并列展示，不自行选择；
最终汇总前暂停，等待责任人确认。
```

## 人工确认点

- 分工是否减少复杂度，而不是制造更多交接。
- 不同角色是否读取了必要且最小范围的材料。
- 冲突、重复和缺口是否完整保留。
- 最终结论是否由有权限的责任人确认。

## 输出物与验收标准

- 角色卡、输入输出契约和交接图完整。
- 每项事实和问题能追溯到来源与生成角色。
- 冲突不会被汇总过程静默删除。
- 任一角色失败时，总任务可以停止或局部重试。

## 常见弯路与安全边界

- 角色越多不代表质量越高，优先使用最少角色。
- 多个角色重复读全部材料，会扩大权限和成本。按需分配。
- 汇总 Agent 不能把不一致结果强行写成一个结论。
- 多 Agent 不替代专业责任、审批和人工复核。
