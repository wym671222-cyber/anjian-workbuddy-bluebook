# 第 23 章 把模板、规范与步骤制作成 Skill

当一个任务已经反复成功，下一步不是复制上次的提示词，而是把适用条件、步骤、检查点和配套模板沉淀成 Skill。Skill 是可复用的工作说明，不是绕过制度和审批的自动通行证。

## 适用场景

- 同类任务每月或每周重复出现。
- 执行步骤稳定，但输入材料每次不同。
- 结果有明确模板和验收标准。
- 新手经常在相同步骤出错。

## 任务目标

让不同的人面对同类任务时，能使用同一套入口、步骤、边界和验收方法，并在失败时知道如何停止。

## 输入材料

- 已验证的任务样本和复盘记录。
- 允许公开或内部使用的模板、规范和检查表。
- 常见错误、异常处理和人工确认点。
- Skill 的适用范围、禁止范围和维护责任。

## 制作步骤

1. **收集成功样本**：至少确认一次完整任务真正通过验收。
2. **拆分稳定与变化**：稳定部分写入 Skill，变化部分作为每次任务输入。
3. **写清触发条件**：说明什么时候应该使用，什么时候不该使用。
4. **设计执行流程**：每一步说明输入、动作、输出和暂停条件。
5. **绑定资源**：复用模板、术语表、示例和检查脚本，不把大段资料全塞进说明。
6. **加入人工闸门**：外发、覆盖、审批、结论和批量写入前暂停。
7. **用合成样本测试**：测试正常、缺材料、格式错误和权限不足等情况。

## 合成示例

将“把一份文字说明转换为五页内部汇报大纲”做成 Skill。Skill 规定输入文件、页面结构、数字核对和输出清单，但不内置任何真实公司模板或材料。

### Skill 设计模板

```text
Skill 名称：{名称}
解决的问题：{一个明确问题}
适用场景：{什么时候使用}
禁止场景：{什么时候不能使用}
必要输入：{文件、字段、授权}
执行步骤：{按顺序列出}
人工确认点：{暂停并等待确认的位置}
输出物：{文件和记录}
验收标准：{可检查条件}
失败处理：{停止、重试、回退}
配套资源：{模板、术语表、样例、检查表}
维护信息：{版本、日期、责任人}
```

## 人工确认点

- 方法是否经过真实任务验证，而非只在纸面上成立。
- 配套模板和资料是否允许进入 Skill。
- 触发范围是否太宽，是否可能误用。
- 高风险动作是否都有明确暂停点。

## 输出物与验收标准

- Skill 说明可以由新手独立理解。
- 输入缺失时会停止并列出缺口。
- 输出结构稳定，关键步骤有日志。
- 合成测试覆盖成功、失败和回退路径。

## 常见弯路与安全边界

- 把整本制度塞入 Skill，会难以维护。Skill 只保留流程，资料按需引用。
- 只写成功路径，遇到异常就会乱试。必须定义停止和回退。
- 一次任务就急着固化，容易把偶然做法当标准。先复现再沉淀。
- Skill 不替代正式制度、权限和专业审核；可复用资产包括说明模板、测试样本和版本记录。
