# 第 23 章 通用任务方法库

本章不展示内部项目或具名成功案例，只把经过实践验证的方法抽象为任务卡。选择方法时先看输入和验收，不要因为名称相似就照搬。

## 方法一：信息候选池

**适用任务**：政策跟踪、行业资讯、专题材料、知识分享候选。

- 输入：主题、时间、来源、排除项和筛选标准。
- 步骤：收集候选、去重分类、人工筛选、事实复核、确认后成稿。
- 验收：每条信息有来源和日期，候选与发布稿分开。
- 边界：不自动发布，不保留受限内容。

## 方法二：统一格式交付

**适用任务**：工作总结、专题汇报、项目汇报、培训材料。

- 输入：内容材料、读者、模板要求和交付格式。
- 步骤：提纲、样页、格式映射、内容生成、数字核对、交付检查。
- 验收：结构完整，格式一致，数字可回查。
- 边界：不公开内部模板，不把版式统一等同于内容正确。

## 方法三：多人计划与资料台账

**适用任务**：工作组计划、活动排期、任务分工、资料共享。

- 输入：目标、里程碑、角色、任务、截止时间和共享范围。
- 步骤：拆分任务、明确依赖、建立台账、更新状态、关闭事项。
- 验收：每项任务有责任角色、时限、成果和状态。
- 边界：不展示真实人员、群聊、内部链接和协作记录。

## 方法四：结构化数据与图表

**适用任务**：问卷、业务统计、意见反馈、分类比较和管理看板。

- 输入：数据副本、字段说明、分析问题和隐私要求。
- 步骤：质量检查、口径确认、处理表、汇总表、图表、证据结论。
- 验收：数字可复算，图表标明样本和限制。
- 边界：不展示个人、真实部门和小样本详情。

## 方法五：多文档清单与流程梳理

**适用任务**：制度索引、文件比对、资料清单、尽调检查和风险识别。

- 输入：授权文件、问题范围、版本信息和专业边界。
- 步骤：文件清单、事实表、差异或证据矩阵、问题清单、人工复核。
- 验收：结论能回到来源，缺口和假设显式标注。
- 边界：不公开真实目录、业务体系、项目数据和流程细节。

## 合成选择示例

任务是“把十篇公开的办公数字化文章整理成一页专题参考”。如果目标是本周发布，应选择“信息候选池”；如果目标是长期检索，应选择“知识资产”；如果还需要多人分工，再叠加“多人计划与资料台账”。先解决一个目标，不一次拼成超大流程。

### 可复制选择模板

```text
我的任务是：{任务}
输入材料是：{材料}
希望得到：{输出}
验收标准是：{标准}
不能执行：{禁止动作}

请从“信息候选池、统一格式交付、多人计划台账、结构化数据、
多文档清单”中选择最接近的方法，并说明：
1. 为什么适用；
2. 哪些步骤可以复用；
3. 哪些地方必须按本任务调整；
4. 需要哪些人工确认；
5. 先做什么最小样本。
```

## 验收与常见弯路

- 方法必须回答“可以用于什么任务”，不能暗示已经成为公司正式流程。
- 合成样例不能混入真实名称、数据、截图和成果。
- 不把不同方法简单拼接成超大流程；从一个可验收任务开始。
- 每次复用后记录偏差和改进，再决定是否升级为 Skill。
