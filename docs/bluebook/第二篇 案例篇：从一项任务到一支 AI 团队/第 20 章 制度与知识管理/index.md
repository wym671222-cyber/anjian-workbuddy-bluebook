# 第 20 章 制度与知识管理

制度与知识管理的难点不是“搜到一段话”，而是确认文件是否有效、内容来自哪里、适用于什么范围。本章只介绍多文档治理的通用方法，不展示真实制度目录或业务资料。

::: warning 使用边界
WorkBuddy 可以建立索引、抽取条款、比较版本和整理问答，但不能认定制度效力、替代权威解释或决定具体事项适用哪条规定。
:::

## 适用场景

- 为一批制度、规范、操作说明建立可检索目录。
- 比较两个版本的新增、删除和修改内容。
- 围绕一个问题查找相关条款并整理出处。
- 把重复出现的问题沉淀为知识卡和检查清单。

## 任务目标

形成“找得到、看得懂、能追溯、会更新”的知识资产，同时保留文件效力、适用范围和解释权限的人工判断。

## 输入材料

- 经授权使用的文件副本或脱敏样本。
- 文件名称、发布主体、发布日期、版本和状态。
- 本次需要回答的问题与检索范围。
- 输出目录、权限范围和更新责任人。

## 执行步骤

### 第一步：先建清单，不急着总结

为每份文件记录编号、名称、版本、日期、来源、状态和存放位置。重复文件先标记，不自动删除。

### 第二步：建立来源层和知识层

来源层保存原文位置与文件信息；知识层保存摘要、主题、关键词和可复用问答。知识卡必须能回到来源层。

### 第三步：按问题检索

让 WorkBuddy 先给出候选条款和出处，再解释它们之间的关系。没有直接依据的推断应单独标记。

### 第四步：进行版本比较

比较时区分新增、删除、文字修改和结构移动，并说明可能影响；“是否构成实质变化”由业务或制度管理人员判断。

### 第五步：形成更新机制

新文件进入后，更新清单、知识卡和引用关系；旧版本保留历史状态，不静默覆盖。

## 合成示例

假设目录内有三份虚构的《办公用品领用指引》样本，其中一份为旧版。WorkBuddy 可以建立文件清单、比较新旧版本并生成查询卡；文件是否现行有效仍由材料提供者确认。

### 可复制任务模板

```text
请对当前工作目录中的文档建立可追溯知识索引。

本次问题：{希望查询或比较的问题}
文件范围：{目录或文件名}
输出范围：{索引/版本对比/条款检索/知识卡}

请按以下步骤执行：
1. 先生成文件清单，记录名称、版本、日期、来源和状态；
2. 标记重复、缺页、无法解析和状态不明的文件，不删除原件；
3. 提取与问题直接相关的条款并标注文件名、章节和页码；
4. 将原文、摘要和你的推断分开；
5. 对版本差异按新增、删除、修改、移动分类；
6. 输出索引、引用清单、待确认问题和更新建议。

不要判断制度效力，不要把推断写成正式解释。
```

## 人工确认点

- 文件是否完整、真实、现行有效且允许使用。
- 版本和发布主体是否识别正确。
- 检索结果是否遗漏关联文件或上位依据。
- 摘要是否改变原意，适用范围是否准确。
- 对外解释和具体适用结论是否由有权限人员确认。

## 输出物与验收标准

- 文件索引：每份文件有唯一记录和来源位置。
- 版本对比表：差异可定位，不只给笼统总结。
- 条款证据包：答案、原文、出处和待确认问题分开。
- 知识卡：说明适用问题、来源、更新时间和责任边界。

## 常见弯路

- **把搜索结果当答案**：命中关键词不代表适用。必须查看上下文和文件状态。
- **只留摘要不留来源**：后续无法复核。所有知识卡都要保留出处。
- **新版本直接覆盖旧版**：变化原因和历史依据会丢失。使用版本关系而不是覆盖。

## 安全边界

- 不公开真实制度清单、内部文件目录和业务流程细节。
- 不扩大文件访问权限，不把受限材料上传到未批准服务。
- 不自动删除重复文件或替换现行版本。
- 不把 AI 摘要作为制度原文或权威解释引用。

## 可复用资产

- 文件元数据清单。
- 版本对比模板。
- 条款证据包模板。
- 知识卡模板。
- 更新与失效检查清单。
