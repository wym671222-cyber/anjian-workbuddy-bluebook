# 第 12 章 文件、版本、终稿与归档治理

文件整理不是“把桌面变干净”，而是保证材料找得到、版本分得清、原件不被误改、成果能够交接。

## 适用场景

- 整理一个项目或专题工作目录。
- 区分草稿、审核稿、定稿、签署版和归档版。
- 批量检查重复文件、异常命名和缺失材料。
- 迁移或交接文件前形成可核对清单。

## 任务目标

在不破坏原始文件的前提下，建立清楚的目录、版本关系、文件清单和处理日志。

## 输入材料

- 明确的工作目录和只读原始目录。
- 文件分类、命名和归档要求。
- 允许移动、复制或重命名的范围。
- 交接对象、截止时间和验收清单。

## 执行步骤

1. **只读预检**：统计文件类型、数量、大小、日期和异常，先不移动。
2. **识别版本**：根据文件名、修改时间和内容摘要建立版本关系，状态不明的进入待确认区。
3. **设计目录**：按任务而不是按软件分类，例如“01_任务说明、02_输入材料、03_过程稿、04_审核稿、05_正式成果、99_待确认”。
4. **生成变更预览**：列出原路径、拟操作、新路径和冲突处理，人工确认后再执行。
5. **保留日志**：记录成功、跳过、冲突和失败；迁移后抽样打开文件并核对数量。

## 合成示例

一个虚构专题目录中有 32 个文件，其中同名文件 4 个、状态不明文件 3 个。WorkBuddy 应先生成清单和建议目录，不能根据“final”“最终”等文件名直接删除其他版本。

### 可复制任务模板

```text
请只读检查当前工作目录，暂时不要移动、删除、覆盖或重命名任何文件。

目标：{整理/交接/归档/迁移}
分类规则：{规则或“请先建议”}
正式成果判断依据：{审批、签署、盖章、确认记录等}

先输出：
1. 文件清单和异常；
2. 可能的重复与版本关系；
3. 建议目录；
4. 拟操作预览和冲突处理；
5. 无法判断、需要人工确认的文件。

没有我确认，不执行写入操作。
```

## 人工确认点

- 原始目录是否保持只读。
- 正式版本是否有可靠依据，而非只看文件名。
- 同名冲突、重复文件和敏感材料如何处理。
- 写入、移动、删除和归档是否有明确权限。

## 输出物与验收标准

- 文件清单完整，异常有标记。
- 每项写操作都有预览和确认记录。
- 原件没有被覆盖，失败可以回退。
- 归档后数量、路径和抽样内容核对一致。

## 常见弯路与安全边界

- 只按修改时间认定终稿，会忽略审批与签署依据。
- 一上来批量重命名，容易破坏外部引用。
- 只看命令没有输出，不代表任务失败或成功，应检查日志和文件状态。
- 删除、覆盖和外部归档必须人工确认；可复用资产包括文件清单、目录模板、变更预览和处理日志。
