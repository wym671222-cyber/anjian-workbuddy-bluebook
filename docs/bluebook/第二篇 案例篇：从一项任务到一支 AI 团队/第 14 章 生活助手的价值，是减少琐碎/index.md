# 第 14 章 把模糊需求变成可验收任务

“帮我整理一下”“做得专业一点”“分析这批材料”都不是可以直接执行的任务。小白使用 WorkBuddy 最重要的一步，是把模糊想法变成目标明确、边界清楚、结果可验收的任务卡。

## 适用场景

- 需求只有一句话，不知道从哪里开始。
- 输入材料很多，但不知道哪些有用。
- 反复修改仍不满意，却说不清问题。
- 需要把任务交给同事或以后重复使用。

## 任务目标

任务开始前回答七个问题：为什么做、给谁看、用什么材料、产出什么、何时完成、如何验收、哪些事情不能做。

## 执行步骤

### 1. 让 AI 先提问

要求 WorkBuddy 用不超过八个问题收集任务约束。回答不了的问题标记待确认，不用猜测填满。

### 2. 形成任务卡

任务卡包含背景、目标、读者、输入、输出、步骤、验收、安全边界和截止时间。

### 3. 先交付最小样本

长报告先做一页，批量文件先做三个，复杂分析先给字段和口径。样本确认后再扩大范围。

### 4. 用验收语言反馈

不要只说“感觉不对”。指出哪项验收标准未满足，例如数字无来源、结构缺章节、语气不适合目标读者。

## 合成示例

原需求是“整理一下本月工作”。澄清后变成：读取三个脱敏周报，面向内部复盘，输出一页摘要和待办清单；事实必须标注来源，不补造结果，不外发。

### 可复制任务模板

```text
我有一个初步需求：{一句话需求}。

请先不要执行。用不超过 8 个问题帮助我澄清：
1. 任务目标和使用对象；
2. 输入材料和可用范围；
3. 输出物、格式和篇幅；
4. 截止时间和优先级；
5. 验收标准；
6. 必须保留和禁止修改的内容；
7. 权限、脱敏和外发边界。

根据我的回答生成任务卡，并建议一个最小可验收样本。
没有我确认，不开始批量处理。
```

## 人工确认点

- 目标和读者是否真实明确。
- 输入材料是否完整并有使用权限。
- 输出格式和验收标准是否可以检查。
- 样本是否通过，是否允许扩大范围。

## 输出物与验收标准

- 任务卡没有“整理好一点”之类不可检查的要求。
- 每项输出都有格式、范围和质量标准。
- 缺失信息、风险和禁止动作单独列出。
- 样本确认记录可作为后续修改依据。

## 常见弯路与可复用资产

- 一句话直接开工，通常导致返工。先澄清再执行。
- 一次处理全部材料，问题会批量放大。先做最小样本。
- 反馈只说“不够专业”，AI无法定位。把反馈写成验收差距。
- 可复用任务卡、澄清问题清单、样本确认单和验收表。
