# 第 13 章 安全远程协作与故障排查

远程入口适合查看状态、补充指令和确认低风险步骤，不适合绕过公司终端管理或把重要操作变成无人值守。

::: warning 使用边界
仅使用公司批准的账号、设备、网络和远程方式。涉及文件外发、系统写入、删除覆盖、审批和账号权限的操作，必须回到受控环境并由有权限人员确认。
:::

## 适用场景

- 离开电脑后查看任务是否完成。
- 补充非敏感说明或确认一个低风险步骤。
- 读取脱敏输出摘要和错误信息。
- 远程停止异常任务并保留日志。

## 任务目标

让远程操作做到身份可确认、权限最小化、范围可限制、过程可追踪、异常可停止。

## 输入材料

- 经批准的远程入口和账号。
- 指定的工作目录、允许动作和禁止动作。
- 任务负责人、确认人和停止条件。
- 日志位置与恢复方式。

## 执行步骤

1. **本地准备**：先在电脑前测试登录、目录和最小权限。
2. **限制范围**：只开放当前任务需要的目录和工具。
3. **先做小样本**：用脱敏文件验证读取、处理和输出路径。
4. **远程只做必要动作**：优先查看、暂停和确认，不远程扩大权限。
5. **分层排查**：依次检查设备在线、网络、账号、权限、工作目录、进程和输出日志。
6. **完成后收口**：退出会话、关闭临时权限并核对输出。

## 合成示例

一项文档转换任务运行时间较长。远程端只查看进度和错误摘要；如果输出目录异常，则停止任务，回到电脑前检查路径，而不是远程授予整个磁盘访问权限。

### 可复制任务模板

```text
请在本次远程任务中遵守以下边界：

工作目录：{目录}
允许动作：{查看状态/读取指定文件/生成草稿/停止任务}
禁止动作：外发、删除、覆盖、审批、扩大权限、访问其他目录
停止条件：{错误次数/路径异常/敏感信息/权限不足}

每一步先报告当前状态、准备执行的动作和影响范围；
遇到权限、网络或目录问题时停止，不要尝试绕过安全策略；
结束后输出操作日志、结果位置和未完成事项。
```

## 人工确认点

- 入口、账号和设备是否经过批准。
- 是否只开放了当前任务所需权限。
- 输出是否可以通过远程渠道查看。
- 异常时是否应该停止并转为本地处理。

## 输出物与验收标准

- 状态报告包含任务、进度、错误和下一步。
- 操作日志能说明何时、由谁、做了什么。
- 临时权限已关闭，没有遗留高权限会话。
- 失败原因定位到设备、网络、账号、权限、路径或任务层。

## 常见弯路与可复用资产

- 网络不通时反复重装应用，通常会掩盖真实问题。按层排查。
- 为了省事开放全部磁盘权限，扩大了误操作范围。坚持最小权限。
- 任务没有输出就判断失败，应同时检查进程、日志和目标目录。
- 可复用远程任务边界卡、分层排查表和结束检查表。
