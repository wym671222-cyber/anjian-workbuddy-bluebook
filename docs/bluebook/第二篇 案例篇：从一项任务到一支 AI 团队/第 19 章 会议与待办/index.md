# 第 19 章 会议与待办

会议价值不在于留下多少文字，而在于把结论、责任和下一步变成可以跟踪的任务。本章介绍一种通用辅助方法，不代表公司现行会议或督办流程。

::: warning 使用边界
录音、转写和参会信息可能包含敏感内容。使用前应确认授权范围；AI 提取的结论、责任人和时限必须由参会人员或会议负责人确认。
:::

## 适用场景

- 会前整理议题、材料清单和待决问题。
- 会中或会后把笔记整理为纪要草稿。
- 从会议材料中提取决定事项、责任人、时限和依赖。
- 形成待办台账、提醒清单和复盘记录。

## 任务目标

让每次会议至少形成四类结果：确认的结论、明确的任务、仍待决的问题、可以回查的依据。

## 输入材料

- 会议主题、时间、参会角色和议题。
- 经授权使用的会议材料、笔记或转写稿。
- 待办台账字段和截止时间口径。
- 需要遵守的纪要模板与共享范围。

## 执行步骤

### 会前：把讨论变成问题

WorkBuddy 先读取材料，输出“需要决定什么、缺什么材料、谁需要准备、预计形成什么结果”。不要用长篇背景占满议程。

### 会后：区分四类信息

将记录拆成事实背景、讨论意见、确认结论、行动任务。意见不能自动升级为结论，模糊表述进入待确认区。

### 建立待办台账

每条任务至少包含任务描述、责任角色、协作角色、截止时间、验收标准、状态和来源。责任人或时限缺失时，不允许自行推断。

### 人工复核并发布

会议负责人确认结论和待办后，再生成纪要发布版。共享前检查参会范围、敏感附件和个人信息。

### 持续跟踪

后续更新只改状态和进展，不改写原始结论。延期、阻塞和范围变化应留下原因与确认记录。

## 合成示例

假设召开一次“办公设备集中盘点准备会”。合成材料包含三项议题和五段笔记。WorkBuddy 可整理出两项确认事项、三项待办和一个待决问题；具体责任人和完成时间仍由会议负责人确认。

### 可复制任务模板

```text
请根据当前目录中的会议材料，生成纪要草稿和待办台账。

会议主题：{主题}
材料范围：{文件名}
纪要模板：{模板文件或“未提供”}
台账字段：任务、责任角色、协作角色、截止时间、验收标准、状态、来源

要求：
1. 区分事实背景、讨论意见、确认结论、行动任务和待决问题；
2. 不把个人观点写成会议结论；
3. 缺失的责任人、时限和验收标准标记“待确认”；
4. 每条结论和任务标注来源材料或记录位置；
5. 先输出预览，等我确认后再生成文件；
6. 不发送消息、不创建外部任务、不覆盖原始材料。
```

## 人工确认点

- 会议记录是否经过授权使用。
- 结论是否真正形成，而不是讨论意见。
- 责任人、完成时间和验收标准是否明确。
- 纪要接收范围和附件是否合适。
- 变更和延期是否得到有权限人员确认。

## 输出物与验收标准

- 纪要草稿：结论明确，意见与结论分开。
- 待办台账：每条任务可执行、可验收、可追踪。
- 待决问题清单：说明缺什么、由谁确认。
- 发布检查表：接收范围、附件和敏感信息已检查。

## 常见弯路

- **按发言顺序复述**：信息很多，但找不到结论。应按议题和结果重组。
- **默认补全责任人和日期**：容易产生错误承诺。缺失项必须显式待确认。
- **纪要发出就算结束**：没有台账和状态更新，任务仍会失联。

## 安全边界

- 未经允许不录音、不转写、不扩大会议材料可见范围。
- 不自动向外发送纪要，不代表负责人创建承诺。
- 不删除原始记录，不用 AI 草稿替代正式确认版本。
- 不把本章方法描述为公司的正式会议制度。

## 可复用资产

- 会前议题卡。
- 纪要结构模板。
- 待办台账字段模板。
- 待决问题清单。
- 发布与复盘检查表。
