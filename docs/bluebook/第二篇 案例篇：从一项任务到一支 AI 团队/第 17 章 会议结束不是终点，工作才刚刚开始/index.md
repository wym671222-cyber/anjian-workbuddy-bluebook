# 第 17 章 结构化数据分析与可视化表达

数据分析的目标不是生成更多图，而是让问题、口径、证据和限制更清楚。本章使用合成数据介绍通用方法，不展示真实部门、人员、样本和内部结论。

## 可应用到哪些任务

- 问卷、意见反馈和满意度分析。
- 业务统计、任务进度和质量数据汇总。
- 分类比较、趋势观察和异常检查。
- 制作管理看板或专题汇报图表。

## 任务目标

做到数据来源清楚、口径一致、处理可复现、图表回答问题、结论带证据、限制不隐藏。

## 输入材料

- 原始数据副本和字段说明。
- 分析问题、目标读者和时间范围。
- 指标定义、分组规则和缺失值处理要求。
- 输出格式、隐私限制和最小展示粒度。

## 执行步骤

1. **读取结构**：先报告表名、字段、行数、数据类型、空值和异常，不修改原表。
2. **确认口径**：把指标公式、时间范围、分组规则和排除条件写成分析说明。
3. **生成处理表**：保留原始列，新增清洗列和计算列，记录每一步。
4. **先表后图**：用汇总表核对数字，再选择比较、趋势、构成或分布图。
5. **结论绑定证据**：每条结论引用具体指标和图表，同时说明样本、缺失和不确定性。
6. **隐私检查**：小样本和自由文本可能识别个人，应合并、隐藏或只在受控环境查看。

## 合成示例

使用 60 条虚构的“办公工具体验反馈”数据，分析使用频率、主要问题和改进建议。图表可以显示总体分布和匿名分组，但不展示个人回答或真实部门。

### 可复制任务模板

```text
请分析当前目录中的表格，先不要修改原文件。

分析问题：{问题}
数据文件：{文件名}
时间范围：{范围}
指标口径：{定义或“待确认”}
分组规则：{规则}
隐私要求：不展示个人信息，不展示可识别的小样本

请先输出字段、数据量、空值、异常和口径问题；
等我确认后再生成处理表、汇总表和图表；
每条结论必须引用证据，并说明样本量、缺失值和限制；
不得根据相关性推断因果，不得补造数据。
```

## 人工确认点

- 数据是否完整、合法且允许用于本次分析。
- 指标、时间、分组和排除规则是否正确。
- 图表是否准确表达数据，没有隐藏低样本或负面结果。
- 结论是否超出证据，是否可能识别个人。

## 输出物与验收标准

- 数据质量报告：字段、空值、异常和处理建议清楚。
- 口径说明：每个指标可以复算。
- 汇总表与图表：数字一致，标题、单位、样本量完整。
- 结论证据表：结论、指标、图表和限制一一对应。

## 常见弯路与安全边界

- 为了图表整齐过滤小组，会造成覆盖不全。应保留全部分组并提示样本量。
- 只看平均值会掩盖分布差异。根据问题补充分布和异常值。
- 把开放文本全部公开会暴露个人，应先脱敏并控制访问。
- 不展示真实部门和个人数据，不把模型推断当管理结论。可复用数据质量表、口径表、图表清单和结论证据表。
