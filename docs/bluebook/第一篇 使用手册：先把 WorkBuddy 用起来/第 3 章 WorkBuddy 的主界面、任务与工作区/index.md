# 第 3 章 WorkBuddy 的主界面、任务与工作区

WorkBuddy 主界面可以理解为三个区域：左侧（侧边栏）管理任务，中间（对话区）下达和追踪任务，右侧（结果区）查看文件、变更、预览和最终产物。

![](assets/001_image_MuLCbdPyDo.png)



## 三个区域分别做什么

| 区域 | 主要用途 | 使用时重点检查 |
|-|-|-|
| 侧边栏 | 新建、搜索、切换和管理任务 | 是否进入了正确任务 |
| 对话区 | 描述需求、补充信息、确认计划 | 目标和约束是否完整 |
| 结果区 | 查看产物、全部文件、变更与预览 | 文件名、路径和改动是否符合预期 |



## 为什么要隔离工作目录

工作目录既是效率设置，也是安全边界。把发票、周报、客户材料混在同一个大目录里，会增加误读、误改和信息串用的风险。

推荐按任务建目录空间。

同时，可以对目录空间的权限进行设置。开启“允许完全访问”后，Agent 可能读写授权目录之外的文件。只有任务确实需要并且范围已经确认时才短时开启，任务结束后及时恢复较低权限。

![](assets/002_image_DtASbQcrto.png)



## 三种工作模式

WorkBuddy 提供三种工作模式：

| 模式 | 中文界面 | 能做什么 | 适合场景 |
|-|-|-|-|
| Ask | 问一问 | 问答、理解和查看，不修改文件 | 先了解资料、确认需求 |
| Craft | 做一做 | 可直接操作本地文件、运行代码及系统指令 | 路径清楚、风险较低的任务 |
| Plan | 想一想 | 先生成计划，确认后再执行 | 多步骤、跨系统、重要文件任务 |

## Agent 高风险模式与高风险操作

风险判断针对 Agent 获得的权限、准备执行的动作和资产当前状态。模型名称只能作为辅助信息，不能单独决定风险等级。

可以用下面的启发式快速判断损失半径：

> 风险暴露 ≈ 权限范围 × 操作不可逆性 × 资产集中度

这个表达只用于快速判断风险优先级，不用于计算事故概率。权限越广、动作越难撤销、重要资产越集中在唯一副本中，越需要降低权限、增加确认和准备恢复路径。

### 哪些情况应按高风险处理

| 风险来源 | 典型情况 | 主要后果 |
|-|-|-|
| 权限范围过大 | 允许完全访问、跨工作区读取、连接器拥有写入权限 | 无关文件或系统被误读、误改 |
| 操作难以撤销 | 删除、覆盖、批量移动或重命名，执行系统命令，修改配置 | 原件、引用关系或运行环境受损 |
| 动作影响外部对象 | 发送邮件、发布内容、提交审批、更新业务记录 | 错误直接传递给客户、同事或正式系统 |
| 资产只有唯一副本 | 正式文件、关键数据或配置没有可恢复副本 | 一次误操作造成无法重建的损失 |
| 自动化扩大影响 | 定时执行、无人值守、批量循环、失败后自动重试 | 小错误快速扩散，难以及时停止 |

### 执行高风险任务前

1. **降低权限**：优先使用 Ask 或 Plan；只授权本次任务所需目录、连接器和账号。
2. **只读预检**：列出准确路径、对象数量、当前状态和计划动作，不立即写入。
3. **确认可恢复**：重要资产先建立独立副本；需要恢复时，应已经知道从哪里恢复、由谁确认。
4. **生成变更预览**：逐项列出原位置、拟操作、新位置、外部影响和冲突处理。
5. **等待明确确认**：确认必须覆盖目标、范围、动作和影响，不能只问一句“是否继续”。
6. **先做小样本**：在脱敏副本或少量对象上验证，再决定是否扩大范围。
7. **保留日志并检查结果**：记录成功、跳过、冲突和失败，抽样打开产物并核对数量。
8. **收回临时权限**：任务完成后关闭完全访问、写入连接和无人值守任务。

### 已经发生异常时怎样处置

1. 立即停止当前 Agent、脚本和自动重试，防止影响继续扩大。
2. 保留日志、时间、命令、目标路径和已产生的结果，不急于反复运行修复命令。
3. 隔离受影响目录或系统，收回不必要的写入权限。
4. 从最近一个可信快照恢复到新的空目录，先校验完整性，不覆盖仍在调查的现场。
5. 由任务责任人核对恢复结果和业务影响，再决定是否替换正式目录、撤回外发内容或恢复自动化。

### 可复制高风险任务提示

```text
这是一项高风险任务。请先进入只读预检，不执行删除、覆盖、批量移动、外发或系统写入。

任务目标：{目标}
授权范围：{目录、账号或系统}
禁止范围：{不能访问或修改的对象}
当前备份：{快照位置、时间和恢复责任人}

请先输出：
1. 准确目标和数量；
2. 权限范围及是否过大；
3. 拟执行动作、不可逆影响和失败停止条件；
4. 变更预览、冲突处理和回退路径；
5. 建议的小样本范围；
6. 需要我逐项确认的内容。

没有明确确认，不执行写入。出现目标不明、路径异常、权限扩大或结果无法验收时立即停止。
```



## 选择不同的模型

默认为自动模式，可以指定你想使用的模型，不同模型积分消耗不同。

![](assets/004_image_OzThbMYn5o.png)

模型名称和能力会随产品更新，不建议长期记住“某个模型只适合某类任务”。更稳妥的做法是按任务选择，并先用同一份脱敏小样本比较结果。

| 任务特征 | 优先关注 | 建议验证 |
|-|-|-|
| 大量文本与长资料 | 上下文长度、事实保持和中文结构能力 | 能否覆盖全部材料并准确引用 |
| 图片、截图或音视频 | 是否支持相应输入模态 | 能否识别关键内容和细节 |
| 表格、代码和文件处理 | 工具调用、格式生成和错误修复能力 | 产物能否打开、计算和复现 |
| 高频简单任务 | 响应速度和使用成本 | 质量达到要求后再批量使用 |
| 高风险专业任务 | 推理深度、证据意识和边界表达 | 只生成辅助材料，交专业人员复核 |

最新模型通常能力更全面，但不代表每项任务都更合适。模型更新后应重新做小样本测试，并以 WorkBuddy 当前界面显示的能力和使用规则为准。
